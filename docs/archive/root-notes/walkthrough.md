# Connect Dise 테스트 가이드

## 개발 환경 실행

```bash
npm install
npm run dev
```

Express 서버(:3000)와 Vite 개발 서버(:5173)가 동시에 실행됩니다.

---

## 1단계: PC 브라우저에서 로비 접속

터미널에 출력된 **Network 주소**를 사용하세요.

```
Network: https://192.168.x.x:5173/
```

> `localhost`로 접속하면 QR코드에 `localhost`가 포함되어 모바일에서 접속이 불가합니다.

로비에서 수록 게임 목록이 카드 형태로 표시됩니다. 원하는 게임을 클릭하면 호스트 화면으로 이동합니다.

---

## 2단계: 모바일 컨트롤러 연결

1. PC와 **동일한 Wi-Fi**에 연결된 스마트폰으로 화면의 **QR코드를 스캔**합니다.
2. 자체 서명 인증서 경고가 표시되면 **고급 > 안전하지 않음으로 이동**을 선택합니다.
3. 센서를 사용하는 게임의 경우 **모션 센서 권한을 허용**합니다.
4. **준비 버튼**을 누르면 호스트에 준비 상태가 전달됩니다.
5. 모든 플레이어가 준비 완료되면 게임이 자동으로 시작됩니다.

---

## 게임별 플레이 방법

### 눈치 10단 (2~6명)

1. 각 플레이어는 모바일에서 **1~10 숫자 카드** 중 하나를 선택합니다.
2. 모든 플레이어가 카드를 제출하면 호스트 화면에서 카드가 공개됩니다.
3. 나보다 **낮은 숫자를 낸 플레이어 수**만큼 점수를 획득합니다.
4. 10라운드 후 최종 점수로 순위를 결정합니다.

### 팽이 배틀 (2~6명)

1. 모바일을 **세게 흔들어** 팽이의 회전력(RPM)을 충전합니다.
2. 충전이 끝나면 팽이가 발사되어 호스트 화면의 3D 아레나에 등장합니다.
3. 모바일을 **기울여서** 팽이를 조종합니다.
4. 다른 팽이를 아레나 밖으로 밀어내거나 회전력이 0이 되면 탈락합니다.
5. **마지막까지 살아남은 팽이가 승리**합니다.

### Digit Puzzle (2~4명)

1. 모바일에서 **닉네임을 입력**하고 준비 버튼을 누릅니다.
2. 모든 플레이어가 준비되면 호스트가 **동일한 4×4 셔플 보드**를 전송하고 게임이 시작됩니다.
3. 각 플레이어는 모바일에서 **타일을 탭하거나 스와이프**하여 퍼즐을 풉니다.
4. 호스트 화면에는 각 플레이어의 **진행률 프로그레스 바**가 실시간으로 표시됩니다.
5. **가장 먼저 퍼즐을 완성**한 플레이어가 승리합니다.

### 주사위 (1~6명)

1. 모바일 화면을 **더블 탭**하면 주사위가 날아가는 애니메이션이 실행됩니다.
2. PC 메인 화면에 **3D 주사위가 굴러떨어지는** 물리 연출이 표시됩니다.

## 프로덕션 환경 테스트

```bash
npm run build
node server/index.js
```

`http://서버IP:3000`으로 접속하여 동일하게 테스트합니다.

---

## 주의사항

- **HTTPS 필수**: 모바일 센서(DeviceOrientation/DeviceMotion)는 HTTPS 환경에서만 동작합니다. 개발 시 Vite의 `@vitejs/plugin-basic-ssl`이 자동으로 자체 서명 인증서를 생성합니다.
- **iOS Safari**: 센서 권한은 반드시 사용자 제스처(버튼 클릭) 내에서 요청해야 합니다. SDK가 이를 자동으로 처리합니다.
- **재연결**: 플레이어가 일시적으로 연결이 끊겨도 30초 이내에 복귀하면 기존 상태가 유지됩니다.
