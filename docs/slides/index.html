02 / Product

💡 PM 제안 및 소구포인트 (USP)

모바일-TV 정보 비대칭성의 극대화

기존 잭박스 파티팩(Jackbox) 등의 글로벌 성공 포뮬러를 우리 플랫폼의 경량 웹 아키텍처에 맞게 이식합니다.

  • 비밀 가림막 기술: 각 플레이어는 모바일 개별 세션에서 본인만의 비밀 단어를 은밀하게 확인하여 물리 보드게임의 손맛 구현
  • 물리 기반 단어 플로팅: 힌트 제출 시 TV에 텍스트가 플로팅되어 시각적 즐거움 선사
  • 극도의 간결성: 3인 이상 누구나 브라우저 접속 즉시 5초 만에 룰 학습 완료

🎯 핵심 시장 타겟

  • 보드게임방/엠티 공간의 파티 피플
  • 거실에서 가족 단위로 가볍게 즐길 콘텐츠
  • 웹 기술 기반의 무설치 설치 장벽 제거
03 / Architecture

🛠️ 개발자 설계 및 기술 청사진

HostSDK & MobileBaseGame의 조화

플랫폼 코어와 콘텐츠 레이어를 엄격하게 분리하여, 서버 자원 낭비 없이 완벽하게 클라이언트 기반으로 구동되는 시스템을 설계합니다.

  • 호스트(Host): HostBaseGame 상속. lobby ➔ setup ➔ discussion ➔ voting ➔ result 순서로 SPA 화면 전환 제어.
  • 모바일(Mobile): MobileBaseGame 상속. waiting ➔ role-reveal ➔ submit-hint ➔ vote 스크린 렌더링.
  • 보안성 확보: 서버/호스트에서 각 클라이언트로 단어를 개별 유니캐스트함으로써 소스코드 인스펙션에 의한 키워드 유출 방지.

🔌 핵심 통신 방식

  • P2P WebRTC: 모바일 조작 실시간 전송
  • Socket.IO Fallback: P2P 실패 시 서버 투명 릴레이 백업
04 / QA & Verification

🧪 품질 보증 & 엣지 케이스 정복

모바일 특화 환경을 위한 E2E 신뢰성 검증

다양한 기기 및 웹 환경에서 동작하는 보드게임의 신뢰성을 담보하기 위해 예외 처리에 집중적인 QA를 전개합니다.

  • 세션 복구(Rejoin) 테스트: 힌트 제출 도중 브라우저 리프레시 시 상태값 유지 여부 교차 검증
  • P2P 차단 예외 테스트: 방화벽 등으로 P2P 터널링이 막힐 때 소켓 백업 통신의 투명한 전환 확인
  • 동시 조작 분산 부하 테스트: 수십 명의 플레이어가 투표 마감 직전 동시에 터치하여 쏠리는 동시성 이슈 정밀 모니터링

📈 목표 품질 지표

  • 핵심 인수 시나리오 통과율 100%
  • Rejoin 세션 유지 성공률 100%
  • P2P Fallback 투명 지연 150ms 미만
05 / Decisions

📝 PL 의사결정 하이라이트
