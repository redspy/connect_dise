# 🎬 [connect_dise] 플랫폼 개선 및 Pirate's Plunder 게임 구현 결과서

본 문서는 플랫폼 내 모바일/태블릿 반응형 레이아웃 개선 결과와 더불어, **7인 확장 에이전트 협업 파이프라인**을 통해 선정된 신규 소셜 심리 게임 **`Pirate's Plunder (해적의 전리품)`**의 최종 구현 및 검증 결과를 정리한 보고서입니다.

---

## 🛠️ 주요 달성 사항 (Key Accomplishments)

### 1. 호스트 게임판 모바일 반응형 및 스크롤 개선
- **스크롤 제한 조건부 완화**: 모바일 및 태블릿 화면 크기 대응을 위해 `overflow: hidden; height: 100vh;` 잠금을 해제하고 스크롤되도록 처방하여 화면 잘림 현상을 극복했습니다.
- **레이아웃 그리드 유동 전환**: 로비의 QR/설정 2컬럼 배치와 인게임 카드 대시보드를 수직 1열 정렬로 자동 전환되게 함으로써 결함을 해결했습니다.

### 2. 신규 게임 'Pirate's Plunder' (해적의 전리품) 완벽 구현
- **로비 배너 연동**: AI 이미지 제너레이터로 생성한 고화질 카툰 스타일 배너 적용.
- **게임 엔진 설계**: 3인 이상 홀수 플레이어 참여 시 1명을 망보기(Lookout)로 자동 배정하고, Lookout이 5라운드 중 최대 1회만 되도록 라운드 로테이션 가드 적용.
- **모바일 조작계**: 
  - 키보드 입력을 완전히 차단하여 뷰포트 버그 원천 방지.
  - "Split 🤝" 과 "Steal 🏴‍☠️" 선택 구조. 오작동을 차단하기 위해 훔치기 버튼 상단에 **오른쪽으로 밀어 잠금 해제(Slide to Unlock)** 래치 트랙 및 제스처 핸들러 탑재.
- **진동 피드백**: 선택 확정(약한 진동), 훔치기 잠금 해제(짧은 이중 진동), 공동 분배 성공(삼중 경쾌 진동), 약탈 당함(길고 강한 충격 진동), 훔치기 성공(잭팟 웅장 진동), 공동 훔치기 실패(이중 중량 충격 진동) 구현.
- **재연결 복구**: 라운드 도중 새로고침/튕김 발생 시, Host가 `rejoinState` 이벤트를 유니캐스트하여 파트너 정보, 라운드 번호, 기존 투표 완료 상태 등을 그대로 복구.
- **AFK 미제출 방지**: 30초 시간 초과 시 미제출자의 결정을 자동으로 "나누기(Split)"로 전환하는 안정성 가드 연동.

## 🧪 검증 결과 (Verification Results)

- **Vite Production Build (`npm run build`)**: 번들러 컴파일이 100% 성공하였습니다. 새로 추가된 `piratePlunderHost` 및 `piratePlunderMobile` 엔트리들이 이상 없이 빌드되었습니다.

### 2. Playwright E2E 통합 테스트 통과

- **수행 시나리오**:
  1. Alice, Bob, Charlie 3인의 플레이어 입장 검증
  2. 홀수 조건에 의한 1명 Lookout 지정 및 2명 해적 매칭 검증
  3. 라운드 1에서 Split 대 Steal(드래그 앤 드롭 래치 해제 포함) 선택 및 점수 정산 검증
  4. 라운드 2~5에서 Lookout 로테이션 및 Split/Split 진행 검증
  5. 5라운드 완수 후 최종 리더보드 시상대 렌더링 검증
  6. Restart 버튼 클릭 후 대기방 복원 검증
- **결과**: `1 passed (1.3m)` (100% 성공)
