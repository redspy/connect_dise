## 1. 기획 보강 의도 및 2.0 소구포인트 (USP)

### 💡 기획 보강 의도
기존의 그림 릴레이 게임은 훌륭한 뼈대를 갖추고 있으나, 그리는 과정이 TV 대화면에 노출되지 않아 대기자들이 지루해지고, 글을 읽지 못하는 유치원생들이 단어 추정 턴에서 완벽하게 소외되는 장벽이 있었습니다. 또한, 모바일 화면을 가로/세로로 회전할 때 캔버스 그림이 찌그러지는 왜곡 현상이 있어 사용자 몰입감을 저해했습니다. 

그림 릴레이 2.0은 이러한 한계를 완벽히 극복하고, 결과 발표 이후의 SNS 공유를 원터치로 고도화하여 플랫폼의 킬러 타이틀로 격상시키고자 합니다.

### 🌟 2.0 소구포인트 (USP)
1.  **실시간 드로잉 생중계 (Live Stroke Streaming)**: 플레이어가 모바일 화면에 그리는 붓의 터치와 궤적이 TV 화면에 60fps에 가까운 속도로 실시간 동적 복원되어, 관전자들에게 마치 스포츠 중계를 보는 듯한 박진감을 제공합니다.
2.  **이모지 드롭 하이브리드 단어 턴 (Emoji-Drop Turn)**: 글씨 타이핑이 어려운 미취학 아동을 위해 폰 화면 아래에 귀여운 감정, 동물, 사물, 음식 등의 **이모지 아이콘 팔레트**를 제공합니다. 드래그 앤 드롭으로 이모지를 조립하여 단어 유추 턴을 직관적으로 돌파할 수 있습니다.
3.  **수학적 종횡비 잠금 (Aspect Ratio Lock)**: 캔버스를 4:3 비율로 잠그고 기기 회전 시 CSS 레터박스를 채워, 폰을 어떻게 돌려도 그림 찌그러짐이나 왜곡이 100% 발생하지 않는 미학적 정교함을 제공합니다.
4.  **원터치 모바일 SNS 명작 공유 (QR Web Share)**: TV 최종 화면의 QR 코드를 찍으면 모바일 브라우저에 액자식 합본 카드가 렌더링되며, 폰의 네이티브 **Web Share API**를 기동해 터치 한 번에 인스타그램 스토리나 카카오톡으로 작품을 자랑할 수 있습니다.

## 2. 2.0 핵심 게임 루프 및 페이즈 전환

```
[ 로비 단계 ] ➔ 데모 플레이 실행 가능 (가상 봇 3인의 실시간 그리기 동적 시뮬레이션)
      ↓
[ 시작 및 인트로 ] ➔ TV: 3D 클레이 동물 캐릭터들의 유쾌한 안내 ➔ 3초 카운트다운
      ↓
[ 1라운드: 실시간 그리기 ] ➔ 모바일 드로잉 ➔ TV: 획(Stroke) 실시간 60fps 생중계 노출
      ↓
[ 2라운드: 단어 및 이모지 유추 ] ➔ 이전 그림 수신 ➔ 모바일: 타이핑 또는 이모지 드롭 조합 제출
      ↓
[ 결과 및 갤러리 합성 발표 ] ➔ 체인별 오해의 역사 순차 스크롤 공개
      ↓
[ 최종 릴리즈 및 공유 ] ➔ TV에 결과 QR 노출 ➔ 폰 스캔 ➔ 네이티브 Web Share 자랑하기 ➔ 로비 복귀
```

## 3. 화면별 2.0 세부 스펙 및 데이터 전환

### 1) 호스트 (TV) 대화면 스펙
- **로비 (`data-phase="lobby"`)**:
  - `min-players="3"`과 `title` 속성을 수용하여 배너와 가이드 자동 정렬.
  - "데모 플레이 실행" 버튼 제공. 데모 구동 시 봇 3인의 드로잉 궤적 패킷을 로컬 Fast-Forward 가동하여 TV 화면에 실시간 선이 슥슥 복원되는 attract 연출 제공.
- **게임 진행 (`data-phase="game"`)**:
  - 상단: 제한시간 타이머 바 (제한시간 설정 값 연동)
  - 중앙: 플레이어 수만큼 분할된 실시간 캔버스 그리드. 플레이어들이 모바일로 긋는 마우스/터치 드래그 궤적 패킷을 P2P 수신하는 즉시 실시간 복원 렌더링.
  - "Alice가 붓을 긋는 중...", "Bob이 색상을 바꾸는 중..." 등 동적 비주얼 텍스트 게이지 노출.

### 2) 모바일 (컨트롤러) 스펙
- **그림 그리기 (`data-screen="draw"`)**:
  - 캔버스는 **고정 4:3 비율**로 강제 제한. 외곽은 검은색 레터박스로 채워 찌그러짐 예방.
  - 마우스/터치 드래그 시 Normalized 좌표 `{ x, y, isDrawing, color }`를 15ms 주기로 P2P DataChannel 실시간 송출.
- **단어/이모지 유추 (`data-screen="word"`)**:
  - 중앙: 이전 플레이어의 그림 카드 노출.
  - 하단 입력부: 일반 키보드 타이핑 입력창과 함께, 그 아래에 **감정, 동물, 사물, 음식 등의 이모지 버튼 리스트** 제공.
  - 이모지 탭/드롭 시 입력 필드에 자동으로 `🐶🚀` 등이 임베드되며 텍스트와 하이브리드로 제출 가능.

## 4. 1단계 기획 QA 인수 조건 (Acceptance Criteria)

- [ ] 로비에서 데모 실행 시 가상 봇 3명이 접속하고 실시간 획 드로잉이 TV 미니 캔버스 위에 슥슥 그려져 결과 도출 및 리셋까지 15초 내에 막힘 없이 이행된다.
- [ ] 모바일 기기를 가로에서 세로로 돌리거나 브라우저 크기를 리사이즈해도 기존 그림이 찌그러지거나 픽셀이 터지지 않고 4:3 고정 비율 레터박스로 완벽하게 보존된다.
- [ ] 모바일에서 단어 유추 시 이모지를 탭하여 `🐯🍔` 형태로 하이브리드 단어를 완성하여 성공적으로 전송 및 제출할 수 있다.
- [ ] 결과 발표 단계에서 완성된 최종 카드가 모바일 네이티브 공유 창(Web Share API)을 기동하여 인스타그램이나 카카오톡 링크 전송 프로세스로 문제없이 연계된다.
