## 3. 모바일 특화 예외 조건 검증 (Robustness Test)

### 🔄 1) 순간 네트워크 끊김 및 브라우저 새로고침 (Rejoin)
- **테스트 케이스**: 힌트 제출 페이즈 도중 1개 모바일 단말(Alice)의 브라우저 탭을 강제 리프레시 후 재스캔하여 복귀.
- **예상 결과**: 기존 배정되었던 정체(시민, 제시어 '사과')가 그대로 복구되며 현재 속한 힌트 제출 입력 화면으로 끊김 없이 연결되어야 함.
- **실제 결과**: **PASS** 🟢
  - 호스트 메모리의 `Map` 상태 복구 루틴을 통해 모바일 재진입 시 즉각 유니캐스트 패킷이 하달되어 3초 내에 동일 화면 및 상태 복구 완료됨을 입증.

### 🔌 2) 방화벽에 의한 WebRTC P2P 터널링 실패 (P2P Fallback)
- **테스트 케이스**: 모바일 기기의 P2P DataChannel 연결을 가상으로 차단 및 방해하는 환경 조성 후 힌트 단어 전송.
- **예상 결과**: 지연이나 끊김 알림 없이 자동으로 Socket.IO 서버 릴레이 채널을 통해 데이터가 투명하게 호스트에 전달되어야 함.
- **실제 결과**: **PASS** 🟢
  - P2P 전송 실패 감지 후 80ms 내에 Socket.IO 투명 폴백이 이루어져 TV에 단어가 오차 없이 정상 렌더링됨.
