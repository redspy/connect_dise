# 🕵️‍♂️ 스파이를 찾아라! (Hidden Agent) 기획 명세서 (spec.md)

- **작성자**: [PM] Product Manager 에이전트
- **대상 게임 ID**: `hidden-agent`
- **게임 성격**: 3인 이상 파티형 심리 보드게임 (비대칭 정보 게임)

---

## 1. 기획 의도 및 소구포인트 (USP)

### 💡 기획 의도
스마트폰을 개인 카드 가림막으로 사용하고 거실 TV를 게임 보드로 활용하여, 오프라인에서 물리적인 보드게임을 할 때의 "서로 눈치를 보며 시선을 교환하는 재미"를 디지털 환경에 그대로 복원합니다. 

### 🌟 우리만의 소구포인트 (USP)
1. **완벽한 정보 격리**: 각자의 역할과 단어는 개인 스마트폰 화면에만 노출되므로, 옆 사람에게 들킬 염려 없이 완벽한 심리전을 펼칠 수 있습니다.
2. **실시간 힌트 키워드 플로팅**: 플레이어가 모바일로 단어를 제출하면 TV 화면에 물리 엔진 효과 혹은 애니메이션과 함께 키워드가 둥둥 떠다녀, 시각적 주목도를 높이고 의심의 단초를 제공합니다.
3. **긴장감 넘치는 심문 및 투표 타임**: 모바일로 은밀히 투표하는 동안 TV에는 카운트다운과 투표 완료 현황이 실시간 게이지로 차올라 플레이어들의 손에 땀을 쥐게 합니다.

---

## 2. 핵심 게임 루프 (Core Game Loop)

```
[ 로비 단계 ] ➔ 3인 이상 접속 및 Ready 완료 시 시작
      ↓
[ 역할 배정 ] ➔ 시민(키워드 제공) vs 스파이(다른 키워드 제공 혹은 무제시)
      ↓
[ 키워드 힌트 제출 ] ➔ 모바일에 단어 입력 ➔ TV 화면에 실시간 플로팅 (2라운드 반복)
      ↓
[ 자유 토론 및 투표 ] ➔ 모바일로 은밀한 투표 ➔ TV에 투표 마감 실시간 연출
      ↓
[ 결과 및 승패 판정 ] ➔ 스파이 검거 시 시민 승리 / 생존 시 스파이 승리
      ↓
[ 리셋 ] ➔ 로비로 복귀 혹은 재시작
```

## 3. 호스트 (TV) 화면 흐름 및 레이아웃 (`data-phase`)

호스트 화면은 공통 CSS를 기반으로 한 세련된 다크 테마와 글래스모피즘 카드를 기본으로 사용합니다.

### 1) 로비 (`data-phase="lobby"`)
- **UI 구성**:
  - 중앙의 세션 접속 QR 코드 및 입장 가이드 (`QRDisplay` 컴포넌트 활용)
  - 입장한 플레이어들의 아바타 카드 레이아웃 (플레이어 이름, 준비 상태 표시)
  - 시작/준비 완료 알림 배너
- **연출**: 플레이어가 들어올 때마다 통통 튀는 환영 애니메이션 효과 적용.

### 2) 역할 배정 안내 (`data-phase="setup"`)
- **UI 구성**: 
  - "당신의 휴대폰 화면을 확인하세요! 역할이 배정되고 있습니다..." 라는 대형 메시지 텍스트
  - 은밀한 사운드 힌트 비주얼 효과
- **시간**: 5초간 대기 후 게임 진행 페이즈로 자동 전환.

### 3) 힌트 키워드 제출 및 플로팅 (`data-phase="discussion"`)
- **UI 구성**:
  - 상단: 라운드 표시 및 타이머 바 (제한시간 40초)
  - 중앙 넓은 영역: 플레이어들이 모바일로 제출한 단어 카드들이 떠다니는 캔버스 공간
- **연출**: 단어가 제출될 때마다 해당 플레이어 아바타에서 단어 풍선이 튀어나와 둥둥 떠다니는 연출.

### 4) 투표 진행 (`data-phase="voting"`)
- **UI 구성**:
  - "스파이로 의심되는 자를 지목하세요!" 배너
  - 실시간 투표 게이지 (누가 투표했는지만 표시, 누구를 투표했는지는 비공개)
  - 30초 카운트다운 타이머

### 5) 결과 및 승패 판정 (`data-phase="result"`)
- **UI 구성**:
  - 투표 결과 다수 득표자 공개 연출 (스포트라이트 애니메이션)
  - 최다 득표자의 정체 공개 ("그는 시민이었습니다!" 또는 "스파이를 검거했습니다!")
  - 최종 승리 진영(시민 vs 스파이)의 화려한 승리 배너 및 축하 연출.
  - "처음으로" 버튼 활성화

## 4. 모바일 (컨트롤러) 화면 흐름 및 조작 (`data-screen`)

### 1) 대기 상태 (`data-screen="waiting"`)
- **UI 구성**: "다른 플레이어들을 기다리는 중..." 스피너 애니메이션 표시.
- **역할**: 로비에서 아직 게임이 시작되지 않았거나, 준비 버튼을 누른 후 대기하는 단계.

### 2) 역할/키워드 은밀한 공개 (`data-screen="role-reveal"`)
- **UI 구성**:
  - 탭하여 카드 뒤집기 힌트 ("화면을 터치해서 역할을 확인하세요")
  - 뒤집힌 후: 역할 명시 (예: `시민` / `스파이`)
  - 배정 단어 명시 (예: 시민에겐 `사과`, 스파이에겐 `배` 또는 `?`)
- **주의**: 이 화면은 다른 사람에게 보이지 않게 폰을 가리고 보라는 경고 아이콘 노출.

### 3) 힌트 제출 입력 (`data-screen="submit-hint"`)
- **UI 구성**:
  - 입력창: "이곳에 제시어를 설명하는 단어 하나를 적어주세요." (한 단어 제한)
  - 전송 버튼 (중복 제출 방지 처리)
- **제한**: 제출 완료 시 다시 대기 화면으로 자동 전환.

### 4) 스파이 은밀 투표 (`data-screen="vote"`)
- **UI 구성**:
  - 자신을 제외한 나머지 플레이어들의 닉네임 버튼 리스트 표시
  - 스파이로 의심되는 유저의 버튼을 탭하여 투표 완료
- **제한**: 1인 1표 필수, 시간 초과 시 무효표 처리.

## 5. 예외 상황 처리 및 인수 조건 (Acceptance Criteria)

### 🛡️ 예외 처리 필수 조건
1. **갑작스러운 연결 끊김 및 재연결 (Rejoin)**:
   - 게임 도중 플레이어의 스마트폰 브라우저가 리프레시되거나 끊겼다 재접속되어도 기존 배정된 역할과 키워드가 그대로 복구되어 해당 페이즈로 복귀해야 합니다.
2. **최소 인원 통제**:
   - 최소 3인 이상일 때만 게임 시작이 가능해야 합니다.
3. **시간 초과(Timeout) 대응**:
   - 힌트 제출 시간이나 투표 시간이 초과되면, 제출하지 않은 유저는 공백 혹은 무효 처리 후 자동으로 다음 페이즈로 강제 이행되어 게임이 멈추지 않아야 합니다.

### 📋 인수 조건 (Acceptance Criteria)
- [ ] 3인 플레이어로 로비 접속 및 전체 Ready 시 호스트 화면이 `setup` 페이즈로 전환된다.
- [ ] 모바일 기기별로 1명의 스파이와 N-1명의 시민이 무작위 배정되며 각각 다른 비밀 단어가 표시된다.
- [ ] 플레이어가 모바일로 전송한 힌트 텍스트가 호스트(TV) 화면에 실시간 렌더링된다.
- [ ] 모바일 투표 시 투표 완료 상태가 실시간으로 호스트 화면의 게이지에 채워진다.
- [ ] 스파이 검거 여부에 따라 호스트 화면에 올바른 승리 결과 배너가 출력되고 "리셋" 시 정상적으로 로비로 복귀한다.
