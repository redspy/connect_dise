# PRD: High Unique

- **문서 버전**: v0.2
- **작성일**: 2026-03-08
- **문서 목적**: 대형 공용 화면 + 개인 모바일 컨트롤러 구조에 맞는 숫자 심리전 보드게임의 MVP 제품 요구사항 정의

---

## 1. 제품 개요

**High Unique**는 각 플레이어가 `1`부터 `10`까지의 숫자 카드 10장을 한 장씩 가지고 시작하여, 매 라운드 비밀리에 카드 1장을 제출하는 심리전 보드게임입니다. 공용 화면에는 전체 진행 상황과 결과가 표시되고, 각 플레이어의 모바일 화면은 카드 선택과 아이템 사용을 위한 개인 인터페이스 역할을 합니다.

게임의 핵심은 단순한 “가장 큰 수가 이긴다”가 아니라, **동점이 발생할 수 있는 심리전**과 **라운드 점수의 누적 구조**, 그리고 **제한된 더블 아이템 사용 타이밍**입니다.

---

## 2. 문제 정의

이 플랫폼은 다음 제약과 기회를 동시에 가집니다.

### 제약
- 모바일을 실시간 정밀 조작용 컨트롤러로 쓰기 어렵다.
- 공용 화면과 개인 화면 사이의 역할 분리가 명확해야 한다.
- 짧은 설명만으로도 처음 보는 사람이 참여할 수 있어야 한다.

### 기회
- 개인 모바일 화면을 통해 **비밀 정보**와 **개인 선택**을 자연스럽게 다룰 수 있다.
- 공용 화면은 결과 공개, 점수 경쟁, 리액션 유도에 강하다.
- 턴 기반/동시 입력형 게임은 네트워크 지연의 영향을 덜 받는다.

**High Unique**는 이 구조에 맞춰, 입력은 단순하게 유지하면서도 심리전과 점수 설계를 통해 반복 플레이 재미를 만드는 것을 목표로 한다.

## 3. 제품 목표

### 목표
1. 룰 설명 1분 이내에 플레이 가능해야 한다.
2. 1판이 5~10분 내외로 끝나야 한다.
3. 공용 화면과 모바일 화면의 역할이 자연스럽게 분리되어야 한다.
4. 첫 프로토타입 기준으로 구현 난이도가 낮아야 한다.
5. 매 라운드 결과 공개 순간에 대화와 반응이 터져 나와야 한다.

### 비목표
- 실시간 액션성 제공
- 복잡한 경제 시스템 또는 긴 메타 성장
- 텍스트 입력 기반 자유 행동
- 긴 세션 운영

## 4. 핵심 게임 컨셉

### 한 줄 요약
각 플레이어는 10라운드 동안 숫자 카드 10장을 한 번씩 사용한다. 매 라운드 가장 큰 숫자가 우위에 서지만, **라운드 점수는 “나보다 낮은 숫자 카드를 낸 플레이어 수”**만큼 얻는다. 여기에 각자 3개의 **더블 아이템**을 적절한 타이밍에 사용해 점수를 2배로 만들 수 있다.

### 핵심 재미
- 높은 숫자를 아껴서 결정적인 순간에 쓸지, 초반부터 선점할지 고민하는 재미
- 다른 플레이어가 어떤 숫자를 낼지 읽는 심리전
- 동점 때문에 발생하는 불확실성
- 더블 아이템 사용 시점에서 오는 리스크/리턴 판단

---

## 5. 타깃 유저

- 행사장, 라운지, 체험 부스, 카페, 리조트 등 공용 대형 화면 환경의 사용자
- 게임 숙련도가 낮아도 바로 참여 가능한 콘텐츠를 원하는 사용자

### 2차 타깃
- 스트리머/관전자와 함께 반응형 플레이를 하고 싶은 사용자
- QR 접속 기반의 빠른 회전형 콘텐츠가 필요한 운영자

## 7. 핵심 규칙

### 7.1 라운드 시작
매 라운드마다 모든 플레이어는 아래 2가지를 비밀리에 결정한다.
1. 이번 라운드에 낼 숫자 카드 1장
2. 더블 아이템 사용 여부

### 7.2 카드 사용 제한
- 숫자 카드 `1~10`은 **각 게임에서 한 번만 사용 가능**하다.
- 사용한 숫자 카드는 이후 라운드에서 다시 사용할 수 없다.
- 총 10라운드가 지나면 모든 숫자 카드가 소진된다.

### 7.3 공개 방식
- 모든 플레이어가 제출을 완료하면 숫자 카드를 동시에 공개한다.
- 공용 화면에는 각 플레이어의 제출 카드와 라운드 결과가 표시된다.

### 7.4 라운드 점수 계산
플레이어의 **기본 라운드 점수**는 다음과 같이 계산한다.

> **기본 라운드 점수 = 해당 플레이어가 낸 숫자보다 엄격히 낮은 숫자를 낸 플레이어 수**

동일 숫자는 “낮은 숫자”로 계산하지 않는다.

#### 예시 1
제출 카드: `10 / 8 / 8 / 5`
- `10` 플레이어: 아래에 `8, 8, 5`가 있으므로 **3점**
- `8` 플레이어 A: 아래에 `5`만 있으므로 **1점**
- `8` 플레이어 B: 아래에 `5`만 있으므로 **1점**
- `5` 플레이어: 아래 카드가 없으므로 **0점**

즉, 결과는 **3 / 1 / 1 / 0**이다.

#### 예시 2
제출 카드: `9 / 7 / 4 / 1`
- `9`: 3점
- `7`: 2점
- `4`: 1점
- `1`: 0점

즉, 동점이 없을 때는 자연스럽게 `1등 3점, 2등 2점, 3등 1점, 4등 0점` 구조가 된다.

### 7.5 더블 아이템
각 플레이어는 게임당 **더블 아이템 3개**를 가진다.

- 라운드 카드 제출 시점에 함께 사용 여부를 선언한다.
- 더블 아이템을 사용한 라운드에서, 해당 플레이어의 **최종 라운드 점수는 2배**가 된다.
- 기본 라운드 점수가 `0점`이면 더블을 사용해도 최종 점수는 `0점`이다.
- 더블 아이템은 **사용 선언 즉시 소모**되며, 라운드 결과와 무관하게 반환되지 않는다.
- 한 라운드에 2개 이상 중첩 사용은 불가능하다.

#### 예시 3
제출 카드: `10 / 8 / 8 / 5`
- `10` 플레이어가 더블 사용: `3점 x 2 = 6점`
- `8` 플레이어 A 더블 미사용: `1점`
- `8` 플레이어 B 더블 사용: `1점 x 2 = 2점`
- `5` 플레이어: `0점`

### 7.6 게임 종료
- 10라운드 종료 후 누적 점수가 가장 높은 플레이어가 승리한다.

### 7.7 동점자 처리
게임 종료 시 누적 점수가 같은 플레이어가 2명 이상이면 다음 순서로 타이브레이크를 진행한다.
1. **남은 더블 아이템 수가 적은 플레이어 우선**
   - 더 많은 리스크를 감수한 플레이를 보상
2. **최고 단일 라운드 점수가 높은 플레이어 우선**
3. 그래도 같으면 **공동 우승**

> 참고: 타이브레이크는 구현 편의상 변경 가능하나, MVP에서는 위 규칙을 기본으로 채택한다.

8. 누적 점수 반영

## 9. UX 원칙

### 9.0 플레이어 식별 원칙
- 각 플레이어는 입장 시 **8개의 프리셋 아바타** 중 하나를 선택한다.
- 공용 화면과 모바일 화면 모두 플레이어를 `아바타 + 테두리 색상 + 닉네임` 조합으로 표시한다.
- 아바타는 작은 화면과 멀리서 보는 공용 화면 모두에서 인지하기 쉬운 단순한 실루엣/아이콘 스타일을 사용한다.
- 테두리 색상은 플레이어 간 구분을 돕는 핵심 시각 장치이며, 동일 방 내 중복을 허용하지 않는다.


### 9.1 공용 화면 역할
공용 화면은 “보드”이자 “결과 연출 장치”다.

필수 표시 요소:
- 게임 제목
- 방 코드 / QR 코드(로비)
- 플레이어 목록 및 연결 상태
- 현재 라운드
- 각 플레이어의 제출 완료 여부
- 카드 공개 결과
- 라운드별 획득 점수
- 누적 점수판
- 우승자 발표

### 9.2 모바일 화면 역할
모바일 화면은 “개인 카드판 + 개인 전략 UI”다.

필수 표시 요소:
- 내 닉네임 / 선택한 아바타 / 테두리 색상
- 남은 숫자 카드 목록
- 선택한 숫자 카드 강조
- 더블 아이템 잔여 개수
- 더블 아이템 ON/OFF 토글
- 제출 버튼
- 라운드 결과 요약
- 현재 누적 점수

### 9.3 UX 기준
- 카드 선택은 **큰 버튼** 또는 **카드형 선택 UI**로 구현한다.
- 더블 아이템 사용 여부는 실수 방지를 위해 시각적으로 강하게 구분한다.
- 제출 후에는 수정 가능 여부를 명확히 표시한다.
- 결과 공개는 전원 동시 공개로 연출하여 긴장감을 만든다.

## 10. 기능 요구사항

### 10.1 로비
- 사용자는 QR 코드 스캔으로 방에 입장할 수 있어야 한다.
- 공용 화면은 현재 방 코드를 노출해야 한다.
- 호스트는 게임 시작을 제어할 수 있어야 한다.
- 최소 인원 이상일 때만 시작 가능해야 한다.

### 10.2 플레이어 입장
- 사용자는 모바일에서 닉네임을 입력해야 한다.
- 사용자는 입장 시 **8개의 프리셋 아바타 중 1개**를 선택해야 한다.
- 플레이어 구분은 아바타와 **테두리 색상** 조합으로 이루어져야 한다.
- 동일 닉네임 충돌 시 시스템이 안내해야 한다.
- 플레이어 연결 끊김 상태를 인식해야 한다.

### 10.3 라운드 입력
- 플레이어는 남은 카드 중 1장을 선택해야 한다.
- 플레이어는 더블 아이템 사용 여부를 함께 선택할 수 있어야 한다.
- 제출 완료 전까지는 선택 변경이 가능해야 한다.
- 제출 완료 후에는 수정 불가 또는 호스트 설정에 따른 수정 가능 정책을 적용한다.

### 10.4 점수 계산
- 시스템은 모든 플레이어의 카드 제출이 완료되면 자동으로 점수를 계산해야 한다.
- 점수 계산에는 현재 라운드 카드와 더블 사용 여부가 반영되어야 한다.
- 누적 점수는 라운드 종료 직후 즉시 갱신되어야 한다.

### 10.5 플레이어 식별 표시
- 공용 화면의 플레이어 카드/패널은 아바타와 테두리 색상을 항상 함께 표시해야 한다.
- 결과 공개 화면에서는 `누가 어떤 숫자를 냈는지`를 즉시 알아볼 수 있도록, 숫자 카드 옆에 아바타와 색상 테두리를 노출해야 한다.
- 모바일 화면 상단에는 항상 자신의 아바타와 테두리 색상을 고정 표시해야 한다.

### 10.6 게임 종료
- 10라운드 종료 후 자동으로 최종 결과 화면으로 전환되어야 한다.
- 최종 결과 화면에는 최종 순위, 총점, 주요 통계가 표시되어야 한다.

## 12. 점수 계산 로직

```ts
function calculateRoundScores(submissions: Array<{ playerId: string; card: number; useDouble: boolean }>) {
  const baseScores: Record<string, number> = {}
  const finalScores: Record<string, number> = {}

  for (const current of submissions) {
    const lowerCount = submissions.filter(s => s.card < current.card).length
    baseScores[current.playerId] = lowerCount
    finalScores[current.playerId] = current.useDouble ? lowerCount * 2 : lowerCount
  }

  return { baseScores, finalScores }
}
```

### 계산 규칙 요약
- 같은 숫자는 서로에게 점수를 제공하지 않는다.
- 점수는 순위가 아니라 **내 카드보다 낮은 카드 수**로 계산한다.
- 더블은 기본 점수 계산 후 최종 점수에 곱한다.

## 15. 밸런스 및 디자인 의도

### 의도 1: 큰 숫자가 항상 강하지만, 무조건적인 정답은 아니다
- 큰 숫자는 기본적으로 더 많은 점수를 노릴 수 있다.
- 하지만 숫자는 한 번만 사용할 수 있으므로, 고점을 언제 터뜨릴지 결정해야 한다.

### 의도 2: 동점은 패널티가 아니라 점수 상한을 낮추는 구조다
- `8, 8`처럼 같은 숫자를 내도 완전히 무효가 되지 않는다.
- 둘 다 아래 숫자에서 점수를 얻을 수 있다.
- 따라서 이 게임은 “동점이면 망한다”가 아니라 “동점이면 손해 본다”는 감각을 만든다.

### 의도 3: 더블은 역전과 폭발력을 만든다
- 더블은 최대 3번만 쓸 수 있으므로, 전반/중반/후반 어느 시점에 쓸지 고민하게 만든다.
- 고점 라운드 예측과 저점 방지 판단이 함께 작동한다.

## 18. 향후 확장 아이디어

   - 쉴드: 이번 라운드 최소 1점 보장
   - 스파이: 한 명의 사용 완료 여부 미리 보기
   - 블라인드: 내 카드 공개를 마지막으로 지연

   - 관전자 승부 예측
   - MVP 플레이어 투표
   - 다음 변형 룰 투표

4. **운영 모드**
   - 카페/행사장용 빠른 회전 설정
   - 자동 다음 방 생성
   - 플레이 통계 대시보드

## 19. 오픈 이슈 및 가정

### 본 문서에서 확정한 가정
- 점수 규칙은 예시(`10 / 8 / 8 / 5`)에 맞춰, **내 카드보다 낮은 카드 수**를 기준으로 계산한다.
- 숫자 카드는 `1~10`을 한 번씩만 사용한다.
- 더블은 사용 선언 시 즉시 소모된다.
- 각 플레이어는 입장 시 **8개 프리셋 아바타 중 1개**를 선택한다.
- 플레이어 구분의 기본 방식은 **아바타 + 테두리 색상 + 닉네임** 조합이다.
- 기본 타깃 인원은 4명이며 MVP 지원 범위는 3~6명이다.

### 추후 검토 가능 항목
- 동률 최종 우승 시 공동 우승 허용 여부
- 제출 제한 시간 도입 여부
- 호스트 없이 자동 시작 가능 여부
- 더블 사용 시 시각/음향 연출 강도

## 20. 한 줄 결론

**High Unique**는 “숫자 카드 한 장과 더블 아이템 한 번의 선택”만으로 심리전, 점수 경쟁, 역전 가능성을 만드는 가벼운 멀티스크린 보드게임이다. MVP 구현 난이도는 낮지만, 공용 화면과 개인 모바일 화면의 역할이 또렷해 플랫폼의 핵심 구조를 가장 깔끔하게 드러낼 수 있다.
