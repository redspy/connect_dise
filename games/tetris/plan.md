# Tetris Battle — 게임 기획

## 컨셉

1~4명이 각자의 모바일로 테트리스를 플레이하는 배틀 로얄 게임.
블록이 천장까지 쌓이면 탈락하고, 마지막까지 살아남은 플레이어가 승리.
시간이 지날수록 블록 낙하 속도가 빨라지며, 다른 플레이어가 줄을 클리어하면
내 속도가 강제로 1단계 상승하는 배틀 메커니즘이 있다.

---

## 게임 흐름

```
로비 → QR 스캔 (1~4명) → 닉네임 입력 → 전원 준비
→ 카운트다운 3, 2, 1 → 게임 시작
→ 호스트: 전체 보드 실시간 표시 + 속도 단계 프로그래스
→ 모바일: 각자 테트리스 플레이
→ 블록 천장 도달 시 해당 플레이어 탈락
→ 1인: 100단계 도달 시 클리어 / 멀티: 1명 남으면 승리
→ 결과 화면 → 다시하기/로비
```

---

## 속도 시스템

- **100단계**로 구성
- **5초마다 자동으로 1단계 상승**
- **다른 플레이어가 줄 클리어 시 나의 속도 강제 상승**
  - 1줄 클리어 → 상대방 1단계 상승
  - 2줄 클리어 → 상대방 2단계 상승
  - 3줄 클리어 → 상대방 3단계 상승
  - 테트리스(4줄) → 상대방 4단계 상승
- 이미 100단계이면 속도 상승 없음 (최대 100단계 고정)
- 단계별 낙하 속도 (예시)

| 단계 | 낙하 간격 |
|------|----------|
| 1    | 1000ms   |
| 10   | 800ms    |
| 25   | 600ms    |
| 50   | 400ms    |
| 75   | 200ms    |
| 100  | 50ms     |

> 정확한 수치는 구현 시 조정

---

## 1인 플레이 특별 규칙

- 혼자 입장해도 게임 진행 가능
- 100단계에 도달하면 **클리어** (승리)
- 100단계 이전에 탈락하면 실패
- 줄 클리어에 의한 속도 상승 없음 (상대방이 없으므로)

---

## 다음 블록 미리보기 옵션

- **기본값: OFF**
- 호스트 로비 화면에서 **체크박스로 ON/OFF 설정**
- ON 시: 모바일 게임 화면 우측에 다음 블록 표시
- OFF 시: 다음 블록 미표시 (난이도 상승 효과)
- 게임 시작 시 호스트가 `gameStarted` 메시지에 `showNextPiece` 옵션 포함하여 전송

---

## 호스트 화면

### 로비
- 4코너 QR 코드
- 참가자 목록 (닉네임 + 플레이어 색상)
- **다음 블록 미리보기 체크박스** (기본 OFF)
- 준비 상태 및 시작 버튼

### 게임 중
- 플레이어별 카드 (최대 4개, 2×2 그리드)
  - 플레이어 닉네임
  - **속도 단계 프로그래스 바** (1~100단계, 플레이어 색상)
  - **축소된 테트리스 보드** (실시간 업데이트)
  - 탈락 시 카드 흐리게 처리 + "GAME OVER" 표시
- 게임 경과 시간

### 결과
- 최종 순위 표시 (탈락 순서 기준)
- 다시하기 버튼

## 모바일 화면

### 설정 (setup)
- 닉네임 입력 (localStorage 저장)
- 참여하기 버튼

### 대기 (waiting)
- 닉네임 표시
- 함께하는 플레이어 목록
- 준비하기 버튼

### 카운트다운 (countdown)
- 3, 2, 1 카운트다운 화면

### 게임 (game)

```
┌─────────────────────────────────────┐
│  ██████████████████░░░░  Lv.12     │  ← 속도 프로그래스 + 단계
├──────────────────┬──────────────────┤
│                  │  NEXT            │  ← showNextPiece ON일 때만 표시
│  [테트리스 보드] │  ┌────┐          │
│     10 × 20      │  │블록│          │
│                  │  └────┘          │
│                  │                  │
├──────────────────┴──────────────────┤
│  [←]  [←↓]  [↓↓하드드롭]  [→↓]  [→]  │
└─────────────────────────────────────┘
```

### 조작 방식

| 버튼 | 동작 | 방식 |
|------|------|------|
| **게임 보드 터치** | 블록 회전 | 탭 |
| **← (왼쪽)** | 블록 왼쪽 이동 | 탭 / 누르는 동안 반복 |
| **→ (오른쪽)** | 블록 오른쪽 이동 | 탭 / 누르는 동안 반복 |
| **←↓ (왼쪽 하단)** | 소프트 드롭 | 누르는 동안 |
| **→↓ (오른쪽 하단)** | 소프트 드롭 | 누르는 동안 |
| **↓↓ (하단 중앙)** | 즉시 낙하 (하드 드롭) | 탭 |

### 탈락 (eliminated)
- "GAME OVER" 표시
- 내 최종 순위 표시
- 남은 플레이어 현황
- 관전 안내 (호스트 화면 보기 유도)

### 1인 클리어 (solo_clear)
- "CLEAR!" 표시
- 100단계 도달 성공 메시지

### 결과 (result)
- 내 순위 표시
- 전체 순위 및 생존 시간
- 다시하기 버튼

## 테트리스 게임 로직

### 보드
- **10×20** 격자

### 테트로미노 (7종)
- I, O, T, S, Z, J, L
- 7-bag 랜덤 시스템 (7개를 한 묶음으로 랜덤 셔플하여 순서 보장)

### 회전
- 단순 4방향 회전 (0°, 90°, 180°, 270°)
- **Wall Kick** 적용: 벽/블록과 충돌 시 반대 방향으로 밀어내며 회전 시도
  - 우측 벽 충돌 → 좌측으로 1칸 밀어내고 회전
  - 좌측 벽 충돌 → 우측으로 1칸 밀어내고 회전
  - 밀어내도 공간이 없으면 회전 불가

### 줄 클리어
- 한 줄이 꽉 차면 제거
- 클리어한 줄 수만큼 다른 모든 활성 플레이어 속도 상승
- 본인 속도는 변하지 않음

### 게임오버 조건
- 새 블록 스폰 위치(상단 중앙)에 기존 블록이 있으면 즉시 탈락

## 메시지 프로토콜

| 방향 | type | payload | 시점 |
|------|------|---------|------|
| M→H | `setProfile` | `{ nickname }` | 닉네임 입력 |
| M→H | `boardUpdate` | `{ board: number[][], level, lines }` | 블록 고정마다 (스로틀 100ms) |
| M→H | `linesCleared` | `{ count }` | 줄 클리어 시 |
| M→H | `gameOver` | `{}` | 탈락 시 |
| M→H | `soloClear` | `{}` | 1인 100단계 클리어 시 |
| M→H | `requestRematch` | `{}` | 다시하기 |
| H→All | `playerListUpdated` | `{ players }` | 입장/프로필 변경 |
| H→All | `gameStarted` | `{ showNextPiece: boolean }` | 게임 시작 |
| H→Player | `levelUp` | `{ newLevel }` | 속도 강제 상승 (해당 플레이어에게만) |
| H→All | `playerEliminated` | `{ playerId, rank }` | 플레이어 탈락 |
| H→All | `gameFinished` | `{ rankings }` | 게임 종료 |

## 확정된 사항 요약

| 항목 | 결정 |
|------|------|
| 인원 | 1~4명 |
| 1인 승리 조건 | 100단계 도달 시 클리어 |
| 탈락 조건 | 블록이 스폰 위치까지 쌓임 |
| 속도 상한 | 100단계 고정 |
| 조작 버튼 | 5개 (←, ←↓, ↓↓하드드롭, →↓, →) + 보드 탭(회전) |
| 다음 블록 미리보기 | 기본 OFF, 호스트 로비 체크박스로 ON 가능 |
| 홀드 기능 | 없음 |
