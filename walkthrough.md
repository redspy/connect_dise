# 주사위 게임 (Dice Game) Walkthrough 및 테스트 가이드

## 🚀 플레이 방법 (테스트 가이드)

프로젝트 개발이 1차적으로 완료되었으며, 현재 데모 서버가 실행 중입니다! 아래 순서에 따라 직접 게임을 테스트해 보실 수 있습니다.

### 1단계: 메인 보드 (PC 브라우저) 열기
PC에서 아래 주소 중 하나를 클릭하거나 브라우저 주소창에 복사하여 붙여넣으세요.
*   **로컬 접속 (현재 PC):** [http://localhost:5173/](http://localhost:5173/)
*   **네트워크 접속:** `http://192.168.219.109:5173/` 

접속하시면 **녹색 카지노 천**의 배경과 중앙에 대기 상태 텍스트, 그리고 사방에 4개의 **QR코드**가 나타납니다.

### 2단계: 모바일 컨트롤러 연결
스마트폰을 꺼내 메인 화면에 표시된 4개의 QR코드 중 하나를 카메라 앱으로 스캔하세요. (동일한 로컬 Wi-Fi에 연결되어 있어야 합니다.)

**⚠️ 주의사항 (필독)**
> 로컬 테스트망에서 보안(SSL)을 활성화했기 때문에, 모바일 브라우저 진입 시 **"안전하지 않은 웹 페이지" (Your connection is not private)** 경고가 나타납니다. 
> 화면 하단의 **'고급(Advanced)'**을 누르고 **'안전하지 않음으로 이동 (Proceed to 192.168...)'**을 선택해 접속해야 합니다.

1.  모바일 화면에서 **"Grant Permission & Play"** 버튼을 눌러 모션 센서(자이로스코프/가속도계) 권한을 허용해주세요. (iOS 환경 등에서 보안 상 필수입니다.)
2.  권한이 승인되면 상단 네비게이션 좌측의 접속 Dot이 **빨간색(연결 대기)에서 녹색(연결됨)** 으로 바뀝니다. 해당 기기(Player)가 추가되었음을 메인 보드 화면의 텍스트가 갱신되며 알려줍니다.

### 3단계: 주사위 던지기!
1.  **흔들기 (Shake)**: 모바일을 가볍게 흔들면 센서 데이터가 라우팅되고 콘솔상 감지됩니다.
2.  **더블 탭 (Double Tap)**: 스마트폰 화면을 **두 번 연속 터치**하면, 모바일 화면 내의 주사위가 위로 날아가는 슈팅 이펙트 애니메이션이 실행됩니다!
3.  **결과 확인**: 슈팅 이벤트가 Socket.io망을 타고 전달되어, **PC 메인 화면의 빙판(보드판)에 3D 주사위 2개(`2d6`)가 굴러 떨어지는 물리 애니메이션 연출**을 감상할 수 있습니다.

---

### 기술 데모 주요 구현 포인트
*   **비동기 Socket 통신**: PC 브라우저가 접속하는 순간 Host의 고유 `Session ID`가 발급되며, 모바일 기기는 QR을 통해 해당 세션 룸으로 직행하게 설계하였습니다. 이를 통해 여러 대의 PC가 각각의 게임 룸을 동시에 안정적으로 열고, 참가자를 받을 수 있습니다.
*   **모바일 센서 대응**: `DeviceOrientation` 권한 요청 모달 설계 및 iOS 보안 정책에 맞춘 유저-인터랙션(클릭) 기반 권한 획득 파이프라인.
*   **3D 주사위 최적화**: 화면을 무겁게 차지하는 Three.js 전체 구성요소를 짜는 대신, 가볍게 최적화된 `@3d-dice/dice-box` 라이브러리를 CDN에서 직접 Import 하여 빌드 퍼포먼스를 최대로 유지하였습니다.
*   **UI/UX 디자인**: Vanilla CSS로 작성된 카지노 느낌의 Green Felt 텍스쳐 아트.

> [!WARNING]
> **HTTPS / 센서 접근 제약**
> 현재 로컬 개발망(`192.168.x.x`) 테스트 시, 일부 엄격한 모바일 브라우저(특히 iOS Safari)는 `https://`가 아닐 경우 DeviceOrientation 기능을 막아둘 수 있습니다. 
> 이 경우 `ngrok`과 같은 터널링 유틸리티를 활용하거나 안드로이드 디바이스로 1차 확인을 부탁드립니다. 더블 터치 던지기는 터치 기반이므로 센서 권한 유무와 무관하게 동작합니다!
